# Contributing to TealTiger Landing Page

Thank you for your interest in contributing to the TealTiger landing page! We welcome contributions from the community.

## 🚀 Getting Started

1. **Fork the repository** on GitHub
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/tealtiger-landing.git
   cd tealtiger-landing
   ```
3. **Install dependencies**:
   ```bash
   npm install
   ```
4. **Create a branch** for your changes:
   ```bash
   git checkout -b feature/your-feature-name
   ```

## 💻 Development

Run the development server:
```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) to see your changes.

## 📝 Making Changes

### Content Updates
- Edit `app/page.tsx` for content changes
- Update `app/layout.tsx` for metadata/SEO
- Modify `tailwind.config.ts` for styling

### Code Style
- Use TypeScript for type safety
- Follow existing code formatting
- Use Tailwind CSS utility classes
- Keep components functional and simple

### Testing
- Test on multiple screen sizes (mobile, tablet, desktop)
- Verify dark mode works correctly
- Check all links and buttons
- Test in different browsers

## 🔍 Pull Request Process

1. **Update documentation** if needed
2. **Test your changes** thoroughly
3. **Commit your changes**:
   ```bash
   git add .
   git commit -m "feat: add amazing feature"
   ```
4. **Push to your fork**:
   ```bash
   git push origin feature/your-feature-name
   ```
5. **Open a Pull Request** on GitHub

### PR Guidelines
- Provide a clear description of changes
- Reference any related issues
- Include screenshots for UI changes
- Ensure the build passes (`npm run build`)

## 🐛 Reporting Bugs

Found a bug? Please open an issue with:
- Clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable
- Browser/device information

## 💡 Suggesting Features

Have an idea? Open an issue with:
- Clear description of the feature
- Use case and benefits
- Mockups or examples (if applicable)

## 📋 Types of Contributions

We welcome:
- **Bug fixes** - Fix typos, broken links, styling issues
- **Content improvements** - Better copy, clearer explanations
- **Design enhancements** - UI/UX improvements
- **Performance optimizations** - Faster load times
- **Accessibility improvements** - Better a11y support
- **Documentation** - Improve README, add guides
- **Translations** - Multi-language support (future)

## ✅ Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Focus on what's best for the community
- Show empathy towards others
